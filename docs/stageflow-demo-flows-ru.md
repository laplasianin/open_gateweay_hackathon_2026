# StageFlow Demo — Пользовательские флоу

## Что это

Платформа для организаторов массовых мероприятий. Управляет приоритетным доступом к сети для стаффа и VIP, отслеживает плотность толпы, координирует экстренную помощь. Всё на карте в реальном времени.

## Кто пользуется

- **Организатор** — сидит за ноутбуком, видит дашборд с картой
- **Стафф** (охрана, медики, логистика) — смотрит в телефон, видит свой статус
- **VIP-посетитель** — смотрит в телефон, получает буст интернета
- **Обычный посетитель** — может нажать SOS если плохо

---

## Экраны

### Дашборд организатора (ноутбук)

Главный экран демо. Что на нём:

- **Карта** площадки с цветными зонами (сцены, фудкорт, медпункт, VIP, входы/выходы)
- **Маркеры** стаффа и VIP — двигаются по карте в реальном времени
- **Панель стаффа** — список людей, у кого сейчас активен приоритетный интернет
- **Лента событий** — "Juan Garcia вошёл в Main Stage A", "QoD активирован", "SOS от посетителя"
- **Зоны плотности** — зоны меняют цвет: зелёный = нормально, красный = переполнено
- **Выбор ивента** — переключение между Primeweaver Sound и World Cup

### Экран стаффа (телефон)

- Имя, роль (Security / Medical / Logistics)
- Статус сети: "Priority Network: Active" (зелёный) или "Normal" (серый)
- В какой зоне сейчас находится
- Для медиков: алерт "Emergency! 200m от вас" когда кто-то жмёт SOS

### Экран посетителя (телефон)

- Статус: "Internet Boost: Active (14:32 remaining)" для VIP
- Большая красная кнопка SOS
- Для обычного посетителя: просто кнопка SOS, без буста

## Flow 1: Приоритетная сеть для стаффа

```
Стафф идёт на площадку
        ↓
Входит в зону (Main Stage A)
        ↓
Система автоматически детектит: "стафф в зоне"
        ↓
Вызов Nokia QoD API → приоритетный интернет активирован
        ↓
На дашборде: маркер стаффа стал синим, в логе "QoD activated"
На телефоне стаффа: "Priority Network: Active"
        ↓
Стафф выходит из зоны
        ↓
Система снимает приоритет → Nokia QoD API → сессия удалена
        ↓
На дашборде: маркер стал серым, в логе "QoD deactivated"
На телефоне: "Normal"
```

Стафф ничего не делает руками — всё автоматически по геозоне.

## Flow 2: VIP-буст

```
VIP-посетитель входит на площадку
        ↓
Входит в VIP Area
        ↓
Система активирует 15-минутный буст через Nokia QoD API
        ↓
На дашборде: маркер VIP со звёздочкой, таймер "15:00"
На телефоне VIP: "Internet Boost: Active (15:00 remaining)"
        ↓
Таймер тикает. VIP может ходить куда угодно — буст на 15 минут
        ↓
15 минут прошло → буст снимается автоматически
```

Отличие от стаффа: стафф получает приоритет пока в зоне, VIP — на фиксированное время.

## Flow 3: Плотность толпы (фейк для демо)

```
Симуляция идёт 30 секунд
        ↓
Зона Main Stage A становится жёлтой → "Crowd: HIGH"
        ↓
Ещё 15 секунд → зона красная → "Crowd: CRITICAL"
        ↓
В ленте событий: "AI Alert: Main Stage A overcrowded — recommend staff reinforcement"
        ↓
Организатор видит проблему на карте
```

Под капотом: просто таймер меняет цвет зоны. Но на демо выглядит как реальная аналитика.

## Flow 4: SOS / Экстренная помощь

```
Посетителю плохо → открывает телефон → жмёт красную кнопку SOS
        ↓
Система получает координаты посетителя
        ↓
Ищет ближайшего медика на карте
        ↓
Находит Dr. Maria Lopez (200м)
        ↓
Активирует QoD буст медику (чтобы связь точно работала)
        ↓
На дашборде:
  - Красный маркер "SOS" на карте
  - Линия от медика к пострадавшему
  - В логе: "Emergency: medic dispatched"
        ↓
На телефоне медика:
  - Алерт "Emergency! Patient 200m from you"
        ↓
Медик приходит → организатор нажимает "Resolve" → инцидент закрыт
```

## Как выглядит демо на звонке

1. Ведущий открывает дашборд на ноутбуке, шарит экран
2. "Вот наша площадка Primeweaver Sound 2026 в Барселоне, 12 зон"
3. Нажимает "Start Simulation" — маркеры начинают двигаться
4. "Видите, Juan Garcia входит в зону Main Stage A — система автоматически даёт ему приоритетный интернет через Nokia QoD"
5. "А вот Anna Berg, VIP-гость, входит в VIP Area — получает 15-минутный буст"
6. Открывает телефон (или вторую вкладку): "Вот что видит стафф — Priority Network Active"
7. "Обратите внимание, Main Stage A стала красной — система детектит переполнение и алертит"
8. Открывает телефон посетителя, жмёт SOS: "Вот экстренный сценарий — система нашла ближайшего медика, дала ему буст, показала маршрут"
9. Переключает на World Cup: "Та же платформа, другой тип ивента — стадион"
10. "Вопросы?"

## Что реальное, что симуляция

| Компонент | Статус |
|-----------|--------|
| Карта с зонами | Реальные координаты Барселоны / Мехико |
| Движение стаффа по карте | Симуляция (заскриптованные маршруты) |
| QoD активация/деактивация | Реальный вызов Nokia API (или мок пока нет ключей) |
| Геозона "вошёл/вышел" | Своя логика (point-in-polygon), не Nokia Geofencing API |
| Плотность толпы | Фейк (таймер меняет цвет зон) |
| Поиск ближайшего медика | Реальный расчёт по координатам |
| SOS кнопка | Реальный trigger |
